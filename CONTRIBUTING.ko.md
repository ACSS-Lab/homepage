## 개요

논문 등록, 프로필 수정, 오류 제보 등 홈페이지의 모든 콘텐츠 변경 사항은 GitHub Issue 템플릿을 통해 요청해야 합니다.

관리자가 아닌 멤버의 직접 수정(push)은 허용하지 않습니다.  
작성 후, GM(GitHub Manager)에게 Slack DM으로 알려주시면 빠른 처리가 가능합니다.

---

## 요청 유형

아래 표를 참고해 요청 내용에 맞는 템플릿을 선택하세요.

| 템플릿 | 사용 상황 |
|---|---|
| **Add Paper** | 새 논문을 홈페이지에 등록할 때 |
| **Update Paper Info** | 이미 등록된 논문의 정보가 잘못되었거나 변경이 필요할 때 |
| **Add / Update Member Info** | 신규 멤버 등록, 또는 기존 프로필 수정이 필요할 때 |
| **General Request** | 링크 오류, 버그, 뉴스 등록, 디자인 수정 등 그 외 모든 요청 |

## 요청 제출 방법

1. 이 레포지토리의 [Issues 탭](https://github.com/ACSS-Lab/homepage/issues)으로 이동합니다.
2. **New Issue** 버튼을 클릭합니다.
3. 요청 내용에 맞는 템플릿을 선택합니다. 빈 양식(Blank issue)은 비활성화되어 있으므로 반드시 템플릿을 선택해야 합니다.
4. 제목 필드를 작성합니다. 앞의 접두어(예: `[Paper Add]`)는 자동 입력되어 있으며, 그 뒤에 간략한 설명을 추가합니다.  
   예시: `[Paper Add] BitTP - CVPR 2026`
5. 양식의 필수 항목을 모두 작성합니다. 선택 항목은 생략할 수 있으나, 정보가 많을수록 처리가 빨라집니다.
6. **Submit new issue** 버튼을 클릭해 제출합니다.

## 저자 표기 규칙

논문 요청 시 저자 목록을 입력할 때 아래 기호를 이름 뒤에 붙여 표기합니다.

| 기호 | 의미 | 예시 |
|---|---|---|
| `**이름**` | 랩 멤버 (굵게 표시) | `**Mincheol Kang**` |
| `*` | 제1저자 또는 동등 기여(Equal contribution) | `Mincheol Kang*` |
| `†` | 교신저자(Corresponding author) | `Soojean Han†` |

기호는 조합해서 사용할 수 있습니다. 랩 멤버이면서 공동 제1저자인 경우 `**Mincheol Kang***`로 작성합니다.

*전체 예시:*

```
**Mincheol Kang***, *Gildong Hong*, **Soojean Han†**
```

## 제출 후 처리 과정

- 관리자가 Issue를 검토하며, 추가 정보가 필요한 경우 댓글로 문의할 수 있습니다.
- 수정이 완료되면 관련 커밋을 참조하여 Issue가 닫힙니다.
- 제출 후 요청 내용을 변경해야 할 경우, 새 Issue를 열지 말고 기존 Issue에 댓글로 남겨 주세요.

별도의 처리 기한은 없지만, 논문 등록이나 프로필 수정 등 일반적인 요청은 보통 수일 내에 처리됩니다.

---

## 유의사항

- 논문 추가 요청 시 arXiv, CVF, IEEE Xplore 등 공개 링크를 함께 첨부하면 처리가 빠릅니다.
- 프로필 사진은 가능하면 정사각형으로 잘라서 첨부해 주세요.
- 메인 페이지 오타 등 즉각적인 처리가 필요한 요청은 General Request 템플릿에서 우선순위를 **High**로 선택하고 이유를 간략히 적어 주세요.
- 동일한 내용의 Issue를 중복으로 열지 마세요. 유사한 요청이 이미 있다면 해당 Issue에 댓글로 추가해 주세요.
